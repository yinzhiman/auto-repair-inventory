## 问题与决策

### 前端表单车辆选择逻辑

**情况**：原设计使用 el-select 的 allow-create 来同时支持选择和新建车辆，但车牌号是字符串而 vehicleId 是数字，类型不兼容
**处理**：改为分离设计 - 选择已有车辆用下拉框，新建车辆用单独的输入框
**影响**：用户体验更清晰，避免了类型混乱

### 客户路由缺失

**情况**：前端需要获取客户列表和客户的车辆列表，但后端没有对应 API
**处理**：新增 `/api/customers` 和 `/api/customers/:id/vehicles` 接口
**影响**：无，接口简单直接

## 技术债务

- **工单号并发安全**：当前实现未处理并发调用场景，高并发时可能产生重复工单号
- **前端表单验证**：客户选择和车辆选择的验证规则可以更完善
- **里程数显示**：编辑工单时里程数字段未显示（条件渲染问题）
